# Sub-agent Pipeline Orchestration

## Goal

Ensure every campaign run through Claude Code properly invokes specialist sub-agents in order — `campaign-manager` orchestrates; `outlier-data-analyst`, `ad-creative-brief-generator`, `outlier-copy-writer`, and `outlier-creative-generator` each handle their designated stage with full context.

## Background

### campaign-manager — Orchestrator (not just an entry point)

`campaign-manager` is a **continuous supervisor**, not a one-shot script runner. It:
- Holds shared context (project_id, flow_id, config_name, cohort list, TG labels) and passes it to every sub-agent explicitly
- Tracks which stages completed successfully and which are blocked
- Ensures no context is silently dropped between stage handoffs
- Updates memory after every run with learnings for future runs

### competitor-bot — Parallel weekly process

`competitor-bot` runs **independently every week** (not per campaign run). It:
- Scrapes competitor task/browse pages (Turing, Surge, Handshake, Meta Ads Library, Reddit, Trustpilot)
- Produces a structured **experiment brief** (not just a Slack report) saved to `data/competitor_intel/latest.json`
- `ad-creative-brief-generator` reads `data/competitor_intel/latest.json` at brief generation time to incorporate any validated experiment ideas (new hooks, earnings claims, copy angles) into the current campaign

### Data flow

```
competitor-bot (weekly, parallel)
    → data/competitor_intel/latest.json
    ↓ (read at Stage 8b)
campaign-manager (per run, supervises all)
    → outlier-data-analyst          (Stage 1: Redash data)
    → ad-creative-brief-generator   (Stage 8b: reads latest.json + cohort)
    → outlier-copy-writer           (Stage 8c: receives full brief)
    → outlier-creative-generator    (Stage 8d: receives copy + brief)
    → campaign-manager              (Stage 8e-f: image gen + Drive)
    → campaign-manager              (Stage 9: LinkedIn, live only)
    → outlier-data-analyst          (Stage 10: performance report)
    → campaign-manager              (Stage 11: memory update)
```

**The bug:** Users run `dry_run.py` from Claude Code instead of invoking `campaign-manager`, bypassing all sub-agent specialization and context supervision.

## Stage-by-Stage Sub-agent Map

| Stage | Sub-agent | Input | Output |
|-------|-----------|-------|--------|
| 1 | `outlier-data-analyst` | flow_id, config_name, date_range | df_raw (screening data) |
| 8a | `competitor-bot` | TG category, config_name | competitor intel JSON (weekly, optional) |
| 8b | `ad-creative-brief-generator` | tg_category, angle (A/B/C), cohort.name, cohort.rules, cohort.pass_rate, config_name | structured brief: headline, subheadline, photo_subject, midjourney_prompt, gradient |
| 8c | `outlier-copy-writer` | full brief from 8b | variants[]: [{angle, headline, subheadline, cta, photo_subject}] × 3 |
| 8d | `outlier-creative-generator` | variants from 8c + brief from 8b | refined Gemini prompts × 3 |
| 8e | campaign-manager (Python) | Gemini prompts | PNG files |
