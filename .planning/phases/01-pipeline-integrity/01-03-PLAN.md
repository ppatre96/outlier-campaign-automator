If any is missing, log an error and stop — this is a human-action prerequisite.

Step 2 — Trigger refresh: Call refresh_access_token() directly:
```bash
PYTHONPATH=. python -c "
from dotenv import load_dotenv
load_dotenv()
from src.linkedin_api import refresh_access_token
new_token = refresh_access_token()
print(f'New token: {new_token[:20]}...')
print('Token refresh succeeded')
"
```
This should:
1. POST to `https://www.linkedin.com/oauth/v2/accessToken` with the refresh token
2. Receive a new access token
3. Write LINKEDIN_ACCESS_TOKEN=<new_token> back to .env via _update_env_token()

Step 3 — Verify persistence: Read .env and confirm the new token is there:
```bash
PYTHONPATH=. python -c "
# Re-read .env to get the freshly written token
from dotenv import dotenv_values
vals = dotenv_values('.env')
token = vals.get('LINKEDIN_ACCESS_TOKEN', '')
assert len(token) > 20, f'Token too short or empty: {token[:10]}'
print(f'Persisted token: {token[:20]}...')
print('.env token persistence confirmed')
"
```

Step 4 — Verify API works with new token: Make a simple API call:
```bash
PYTHONPATH=. python -c "
from dotenv import load_dotenv
load_dotenv()
import config
from src.linkedin_api import LinkedInClient
li = LinkedInClient(config.LINKEDIN_TOKEN)
# Simple test: get ad account info
import requests
resp = li._req('GET', li._url(f'/adAccounts/{config.LINKEDIN_AD_ACCOUNT_ID}'))
print(f'API response: {resp.status_code}')
assert resp.status_code == 200, f'API call failed with {resp.status_code}: {resp.text[:200]}'
print('LinkedIn API call succeeded with refreshed token')
"
```

If step 2 fails with RuntimeError (credentials missing or invalid), this becomes a blocker requiring human intervention to re-authenticate the LinkedIn OAuth app.
