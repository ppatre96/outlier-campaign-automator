# Feedback Agent: Creative & Cohort Performance Analysis

## Objective

Implement feedback_agent as the first specialist in the Phase 2.5 feedback loop. This agent analyzes creative and cohort performance metrics from Redash, generates data-driven hypotheses about why certain creatives/cohorts work, and produces a weekly Slack alert flagging underperformers.

**Purpose:** Continuously measure what works and what doesn't so experiment_scientist_agent can decide what to test next.

**Output:** 
- Creative performance analysis JSON (FEED-02 artifact)
- Cohort underperformance alerts (FEED-07 artifact)
- Slack alert text formatted per Outlier vocabulary (OBS-01 prerequisite)

**Class: FeedbackAgent**
- Constructor: `__init__(redash_client: RedashClient)`
- Stores redash_client for query delegation

**Method 1: analyze_creative_performance(days_back=7)**
Input: days_back (int, default 7)
Output: List[dict] with schema: `{angle, photo_subject, cohort_name, ctr, cpa, impression_count, rank_in_cohort, reason_hypothesis}`

Logic:
1. Call redash_client.query_creative_performance(days_back=days_back)
2. Group by cohort_name, angle, photo_subject
3. Rank creatives within each cohort by CTR (ascending to find worst), CPA (descending to find expensive)
4. For top 3 and bottom 3 in each cohort:
   - Generate reason_hypothesis using simple pattern matching:
     * If CPA > median+2σ: "High-cost creative; lower engagement rate in target cohort"
     * If CTR > median+2σ: "Strong engagement hook; consider testing similar angle"
     * If photo_subject is unique in cohort: "Unique visual treatment drives differentiation"
     * Else: "Audience resonance varies; test A/B variants"
5. Return list sorted by cohort_name, then by CPA desc (expensive first for review)

**Method 2: identify_underperforming_cohorts(days_back=7)**
Input: days_back (int, default 7)
Output: List[dict] with schema: `{cohort_name, current_cpa, baseline_cpa, sigma, cpa_z_score, ctr_trend, recommendation}`

Logic:
1. Call redash_client.query_cohort_metrics(days_back=days_back)
2. Calculate baseline_cpa = median CPA across all cohorts (current week)
3. Calculate sigma = std dev of CPA (current week)
4. For each cohort:
   - cpa_z_score = (current_cpa - baseline_cpa) / sigma
   - ctr_trend = (current_week_ctr - prior_week_ctr) / prior_week_ctr
   - Underperforming if: cpa_z_score > 2.0 OR ctr_trend < -0.10 (10% decline)
   - recommendation: "PAUSE" if z_score > 3, "TEST_NEW_ANGLES" if trend < -0.10, else "MONITOR"
5. Return list sorted by cpa_z_score desc (worst first)

**Method 3: generate_slack_alert(underperformers: List[dict], hypothesis_summary: dict) -> str**
Input: 
- underperformers: List from identify_underperforming_cohorts()
- hypothesis_summary: Dict from analyze_creative_performance() (preprocessed to top insights)
Output: Formatted Slack message text

Template:
```
Weekly Feedback: Creative Performance Analysis

Top Underperforming Cohorts:
{for each in underperformers[:3]:}
• {cohort_name} — CPA ${current_cpa:.0f} (baseline ${baseline_cpa:.0f}, +{cpa_z_score:.1f}σ)
  Trend: CTR {ctr_trend:+.1%}
  Recommendation: {recommendation}

Creative Insights:
{for each in hypothesis_summary[:5]:}
• Angle {angle} with {photo_subject} in {cohort_name}: {reason_hypothesis}

React with:
👍 to pause a cohort
🧪 to request A/B test of new angles
```
