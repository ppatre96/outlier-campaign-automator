# Phase 2.5 Plan 04: Reanalysis Loop — User Feedback to New Cohort Discovery

## One-liner

Closed the human-in-the-loop optimization cycle: Slack reactions (👍 pause, 🧪 test) now trigger async ReanalysisOrchestrator that reruns Stage A on fresh screening data, stages new cohorts for campaign scheduling, and persists test variant results for next week's feedback_agent evaluation.

## What Was Built

### Task 1: Reaction Callbacks in slack_alert_handler.py

Extended `src/slack_alert_handler.py` with two async module-level callback functions and updated the `SlackReactionHandler` class to register and invoke them.

**`on_pause_cohort(action_context: dict) -> dict`**
- Handles 👍 reaction: logs pause decision with user + timestamp
- Invokes `ReanalysisOrchestrator.trigger_reanalysis(cohort_to_exclude=cohort, reason='user_pause')`
- Returns `{success, action: 'PAUSE', cohort, triggered_by, next_step}`
- Error handling: reanalysis failure adds `warning` key but returns `success: True` (non-blocking)

**`on_test_new_angles(action_context: dict) -> dict`**
- Handles 🧪 reaction: logs test request with user + cohort
- Boosts priority of pending experiments for this cohort in `ExperimentBacklog` by 50%
- Invokes `ReanalysisOrchestrator.trigger_reanalysis(cohort_to_focus=cohort, reason='user_test_request')`
- Returns `{success, action: 'TEST_NEW_ANGLES', cohort, triggered_by, experiments_boosted}`

**`SlackReactionHandler.__init__()` update:**
- Now registers default callbacks: `{'thumbsup': on_pause_cohort, 'lab': on_test_new_angles}`
- Caller-supplied `callback_registry` overrides defaults

**`handle_reaction_event()` update:**
- Looks up callbacks by emoji name (`thumbsup`, `lab`) directly
- Wraps async callbacks via `asyncio.run()` for sync webhook handler compatibility
- Returns callback result directly (structured `{success, action, cohort, ...}`)

### Task 2: ReanalysisOrchestrator (`src/reanalysis_loop.py`)

New class with 3 public methods + module-level convenience wrappers:

**`ReanalysisOrchestrator.trigger_reanalysis(cohort_to_exclude, cohort_to_focus, reason) -> list`**
- Async: invokes `_run_stage_a_discovery()` on fresh screening data
- Filters: removes excluded cohort, prioritizes focus cohort variants
- Calls `stage_new_cohorts()` for new discoveries
- Persists run record to `data/reanalysis_results.json`
- Returns list of new cohort dicts

**`ReanalysisOrchestrator.stage_new_cohorts(cohorts) -> dict`**
- Compares to existing `data/scheduling_queue.json`
- Skips already-queued cohorts
- Assigns `angle='A'` (financial angle locked decision) to all new cohorts
- Returns `{staged: N, in_queue: total}`

**`ReanalysisOrchestrator.collect_test_results(test_variant_cohorts) -> dict`**
- Queries Redash for test variant vs baseline CTR/CPA
- Calculates delta; classifies as won (>+5%), lost (<-5%), neutral
- Marks experiments completed in `ExperimentBacklog` (FEED-14 closure)
- Persists results to `data/test_variant_results.json`
- Returns `{results: {cohort: {delta, result}}, summary: {won, lost, neutral}}`

**Stub mode:** When `REDASH_API_KEY` is unset, `_run_stage_a_discovery()` returns representative cohort structures so the pipeline remains testable without live credentials.

**Module-level exports:** `trigger_reanalysis()` and `stage_new_cohorts()` sync wrappers for non-async callers.

### Task 3: CLI Script (`scripts/trigger_reanalysis.py`)

Standalone CLI for manual reanalysis triggering:

```bash
# Reanalyze excluding a paused cohort
python scripts/trigger_reanalysis.py --exclude DATA_ANALYST

# Discover variants for a specific cohort
python scripts/trigger_reanalysis.py --focus ML_ENGINEER

# Collect test results for specific cohorts
python scripts/trigger_reanalysis.py --collect-results --cohorts DATA_ANALYST,ML_ENGINEER

# Dry run (print JSON; don't execute)
python scripts/trigger_reanalysis.py --focus ML_ENGINEER --dry-run
```

Arguments: `--exclude`, `--focus`, `--collect-results`, `--cohorts`, `--dry-run`

Output: structured JSON + human-readable cohort table.

## Feedback Loop Closure

```
Slack reaction (👍/🧪)
  → handle_reaction_event()
  → on_pause_cohort() or on_test_new_angles()
  → ReanalysisOrchestrator.trigger_reanalysis()
  → _run_stage_a_discovery()  (Redash or stub)
  → _filter_cohorts()         (exclude/focus)
  → stage_new_cohorts()       → scheduling_queue.json
  → Next Monday: campaign-manager reads scheduling queue
  → Campaigns run with new cohorts
  → collect_test_results()    → test_variant_results.json
  → feedback_agent reads results → next weekly alert
```

## Deviations from Plan

### Auto-adapted: Callback registration approach

**Found during:** Task 1

**Issue:** The plan specified registering callbacks using emoji chars ('👍', '🧪'), but the existing `SlackReactionHandler` used Slack emoji names ('thumbsup', 'lab') as the internal key. The existing `handle_reaction_event()` looked up `emoji_char = '👍' if ... else '🧪'`.

**Fix:** Standardized on emoji names ('thumbsup', 'lab') as the callback dict key throughout — matches the Slack API event format and avoids Unicode lookup inconsistencies. `handle_reaction_event()` now directly looks up `self.callbacks.get(reaction_emoji_name)`.

**Files modified:** `src/slack_alert_handler.py`

### Auto-adapted: Async Stage A uses stub mode when Redash unavailable

**Found during:** Task 2

**Issue:** The plan specified calling `outlier_data_analyst` as an external sub-agent, but no sub-agent invocation mechanism exists in the current codebase (it's Claude Code session-only). The Stage A logic in `stage_a.py` does not exist in the src/ directory.

**Fix:** `_run_stage_a_discovery()` queries Redash directly for fresh cohort data (matching the existing pattern). When `REDASH_API_KEY` is unset, it returns stub cohort structures so the pipeline chain remains testable. This is the correct production approach — `outlier_data_analyst` is the Claude Code session agent, not a Python callable.

**Files modified:** `src/reanalysis_loop.py` (implementation only, no plan deviation)
