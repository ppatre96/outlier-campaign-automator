# Reanalysis Loop: User Feedback → New Cohort Discovery → Scheduling

## Objective

Close the feedback loop by routing user reactions from Slack directly into a reanalysis pipeline. When users pause an underperforming cohort or request new angles, the system reruns Stage A (cohort discovery) on fresh screening data, discovers new cohorts, and feeds them back to the campaign-manager for the next run.

**Purpose:** Enable human-in-the-loop optimization; new cohorts discovered weekly drive continuous campaign expansion.

**Output:**
- New cohorts from fresh screening data analysis
- Campaign scheduling queue fed to campaign-manager
- Test variant results persisted for next week's feedback loop (FEED-14)

**Method 1: trigger_reanalysis(cohort_to_exclude: str = None, cohort_to_focus: str = None, reason: str = 'manual') -> list[dict]**
Input:
- cohort_to_exclude: don't rediscover this cohort (user paused it)
- cohort_to_focus: prioritize discovering variants of this cohort
- reason: logging reason ('user_pause', 'user_test_request', 'weekly_refresh', 'manual')

Logic:
1. Fetch fresh screening data via outlier_data_analyst (invokes external agent)
2. Run Stage A (cohort discovery) on fresh data
3. Filter results:
   - If cohort_to_exclude: remove any cohorts matching that name
   - If cohort_to_focus: prioritize cohorts with high overlap to focus cohort
4. For each new cohort discovered:
   - Call stage_new_cohorts() to queue for scheduling
5. Persist discovery results to data/reanalysis_results.json
6. Return list of new cohorts discovered

**Method 2: stage_new_cohorts(cohorts: list[dict]) -> dict**
Input: cohorts from fresh Stage A analysis

Logic:
1. Compare to existing cohorts in campaign-manager memory
2. For each new cohort:
   - Assign default config + angle (A for financial, per LOCKED decision)
   - Create scheduling entry: {cohort, tg_category, pass_rate, status: 'queued'}
3. Append to campaign-manager.scheduling_queue (or memory equivalent)
4. Log: "Staged {N} new cohorts for scheduling"
5. Return {staged: N, in_queue: total_in_queue}

**Method 3: collect_test_results(test_variant_cohorts: list[str]) -> dict**
Input: list of cohort names that had test variants run

Logic:
1. Query Redash for campaigns WHERE experiment=true AND cohort IN (test_variant_cohorts)
2. Extract performance metrics: CTR, CPA, conversions for test variant vs baseline
3. Calculate delta: (test_ctr - baseline_ctr) / baseline_ctr, similar for CPA
4. For each test variant:
   - Determine result: 'won' (delta > +5%), 'lost' (delta < -5%), 'neutral' (else)
   - Call experiment_scientist_agent.mark_completed(hypothesis_key, result)
5. Persist results to memory for next feedback_agent evaluation
6. Log: "Test results: {N} variants measured; {W} won, {L} lost, {N} neutral"
7. Return {results: {variant_id: {delta, result}}, summary: {won, lost, neutral}}

**Error handling:**
- If outlier_data_analyst agent call fails: log.error() and return empty list (don't abort)
- If Stage A fails: catch RuntimeError and return empty list
- If campaign-manager scheduling queue unavailable: log.warning() and queue to memory instead

- log.info("Stage A discovery: %d new cohorts found", len(new_cohorts))

```python
def main():
    import argparse
    parser = argparse.ArgumentParser(description='Trigger reanalysis and cohort discovery')
    parser.add_argument('--exclude', help='Cohort name to exclude from reanalysis')
    parser.add_argument('--focus', help='Cohort name to focus on for variant discovery')
    parser.add_argument('--collect-results', action='store_true', help='Collect test variant results only')
    parser.add_argument('--cohorts', help='Comma-separated cohort names for result collection')
    parser.add_argument('--dry-run', action='store_true', help='Print what would happen; don\'t execute')
    
    args = parser.parse_args()
    
    orchestrator = ReanalysisOrchestrator()
    
    if args.collect_results:
        # Collect test results for specific cohorts
        cohort_list = [c.strip() for c in args.cohorts.split(',')]
        results = orchestrator.collect_test_results(cohort_list)
        print(json.dumps(results, indent=2))
    else:
        # Trigger reanalysis
        new_cohorts = orchestrator.trigger_reanalysis(
            cohort_to_exclude=args.exclude,
            cohort_to_focus=args.focus,
            reason='manual_cli'
        )
        print(f"Discovered {len(new_cohorts)} new cohorts:")
        for c in new_cohorts:
            print(f"  - {c['name']}: pass_rate={c.get('pass_rate', 'N/A')}")
```
