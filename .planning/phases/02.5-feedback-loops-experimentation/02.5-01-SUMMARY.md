# Phase 2.5 Plan 01: Feedback Agent — Creative & Cohort Performance Analysis Summary

**One-liner:** FeedbackAgent with 2-sigma CPA underperformance detection and pattern-matching creative hypotheses, consuming VIEW.LINKEDIN_CREATIVE_COSTS and VIEW.APPLICATION_CONVERSION via Redash.

## What Was Built

### src/redash_db.py — Two new methods on RedashClient

**query_creative_performance(days_back=7, end_date=None)**
- Queries `VIEW.LINKEDIN_CREATIVE_COSTS` for last N days with >100 impressions filter
- Returns DataFrame: creative_id, creative_urn, cohort_name, angle, photo_subject, impressions, clicks, ctr, spend, conversions, cpa, created_date
- Returns empty DataFrame with correct columns if no data (never None)
- SQL label: `creative-perf-{days}`

**query_cohort_metrics(days_back=7, end_date=None)**
- Queries `VIEW.APPLICATION_CONVERSION` grouped by cohort x week
- Returns 8 weeks of data (current + 7 prior) for trend analysis
- Computes week-over-week CTR delta (trend_indicator) via LAG window function
- Returns DataFrame: cohort_name, week_of, n_impressions, n_clicks, ctr, n_conversions, cpa, trend_indicator
- SQL label: `cohort-metrics-{weeks}`

### src/feedback_agent.py — FeedbackAgent class (365 lines)

**FeedbackAgent.__init__(redash_client: RedashClient)**
- Dependency-injected; all data access goes through redash_client

**analyze_creative_performance(days_back=7) -> List[dict]**
- Groups creatives by cohort, selects top-3 and bottom-3 by CTR within each cohort
- Generates reason_hypothesis via `_generate_hypothesis()` pattern matching:
  - CPA > median+2sigma → "High-cost creative; lower engagement rate in target cohort"
  - CTR > median+2sigma → "Strong engagement hook; consider testing similar angle"
  - Unique photo_subject in cohort → "Unique visual treatment drives differentiation"
  - Fallback → "Audience resonance varies; test A/B variants"
- Returns sorted by cohort_name, then CPA descending (expensive first for review)

**identify_underperforming_cohorts(days_back=7) -> List[dict]**
- Extracts most recent week for each cohort from query_cohort_metrics()
- Computes baseline_cpa = numpy median across all cohorts
- Computes sigma = numpy std(ddof=1) across all cohorts
- Flags cohort if: cpa_z_score > 2.0 OR ctr_trend < -0.10
- recommendation: PAUSE if z > 3.0, TEST_NEW_ANGLES if ctr_trend < -0.10, else MONITOR
- Returns sorted by cpa_z_score descending

**generate_slack_alert(underperformers, hypothesis_summary) -> str**
- Returns "No underperforming cohorts this week." if underperformers is empty
- Formats weekly Slack message per Outlier vocabulary (no banned terms)
- Shows top 3 underperformers with CPA, baseline, z-score, trend, recommendation
- Shows top 5 creative insights from hypothesis_summary
- Reaction prompts: thumbs-up (pause) and test-tube (A/B analysis)

## How to Use

```python
from src.feedback_agent import FeedbackAgent
from src.redash_db import RedashClient

agent = FeedbackAgent(RedashClient())

# 1. Analyze creative performance (7-day rolling window)
creatives = agent.analyze_creative_performance(days_back=7)

# 2. Identify underperforming cohorts
underperformers = agent.identify_underperforming_cohorts(days_back=7)

# 3. Generate Slack alert
alert_text = agent.generate_slack_alert(underperformers, creatives)
print(alert_text)
```

## Key Implementation Details

- **2-sigma threshold:** Uses numpy median (not mean) as baseline to be robust to outliers; uses ddof=1 std for sample statistics
- **7-day rolling window:** Matches Monday 9 AM IST Slack reporting cadence from Phase 2
- **8-week cohort history:** query_cohort_metrics always fetches 8 weeks to ensure prior-week CTR comparison is always available
- **Empty DataFrame guards:** Both query methods return empty DataFrames with correct column names (not None), preventing downstream KeyError
- **Zero-sigma guard:** If all cohorts have identical CPA (sigma=0), z-scores are set to 0.0 (no false positives)

## Blockers

- Redash queries reference `VIEW.LINKEDIN_CREATIVE_COSTS` and `VIEW.APPLICATION_CONVERSION` with specific columns (angle, photo_subject, spend_usd, linkedin_click, activation_day). If these columns don't exist in the views at query time, the SQL will fail at runtime. This is a Redash/Snowflake schema dependency, not a code issue.
- `REDASH_API_KEY` must be set in .env for live execution.
