# Phase 02.5 Plan 06: Sentiment Miner Summary

Standalone src/sentiment_miner.py module that weekly pulls contributor feedback from public review sources (Reddit, Trustpilot, Glassdoor, Outlier Community / Discourse) and internal support channels (Zendesk, Intercom), extracts themed issues via the LiteLLM gateway, and emits data/sentiment_callouts.json as a vocabulary-clean directive feed for ad-creative-brief-generator.

## What was built

- **`src/sentiment_miner.py` (611 lines, new)** — six per-source fetchers (`fetch_reddit`, `fetch_trustpilot`, `fetch_glassdoor`, `fetch_discourse`, `fetch_zendesk`, `fetch_intercom`), an LLM theme extractor (`extract_themes`), and a public `run()` entrypoint that produces both a curated callouts file and a full raw dump for postmortem.
- **`tests/test_sentiment_miner.py` (164 lines, new)** — five unit tests covering FEED-17 fetcher isolation, FEED-18 credential gating for both internal sources, FEED-19 evidence threshold enforcement, and FEED-19 + CLAUDE.md vocabulary scrub. All HTTP and LLM calls are mocked via `monkeypatch`; zero network access during the suite.
- **`.env.example` (new)** — first-of-its-kind in this repo; documents the four new sentiment-related env vars (`SENTIMENT_*`, `ZENDESK_*`, `INTERCOM_ACCESS_TOKEN`) with setup hints for token issuance.
- **`config.py` (modified)** — appended seven sentiment config constants under a new section header. Existing constants untouched.
- **`.claude/agents/ad-creative-brief-generator.md` (modified)** — appended a Sentiment-Driven Copy Inputs section that describes the read contract: schema, "honor at least one Address directive when themes are non-empty" rule, vocabulary requirement, graceful fallback semantics.

Apple App Store and Google Play Store fetchers were intentionally skipped per RESEARCH-V2's verification (2026-04-24) that Outlier operates entirely in-browser with no native mobile app. The module docstring records this rationale.

## How to use

```python
from src import sentiment_miner

# Default: fetch from all six configured sources
result = sentiment_miner.run()
# result = {
#   "callouts_path":   "data/sentiment_callouts.json",
#   "raw_path":        "data/sentiment_raw/2026-04-25.json",
#   "themes_surfaced": <int>,
#   "sources_queried": [...],
#   "sources_skipped": [{"source": "...", "reason": "..."}],
# }

# Or restrict sources (e.g. skip internal during dev without creds)
result = sentiment_miner.run(sources=["reddit", "trustpilot", "discourse"])
```

The brief generator agent reads the resulting `data/sentiment_callouts.json` at brief-generation time and surfaces directives like `"Address: Slow payment release"` directly into the ad copy.

## Key implementation details

- **HTTP fetch pattern** — `_get()` cloned from `src/competitor_intel.py:320-327`, extended with optional `auth=` and `extra_headers=` parameters. Descriptive User-Agent (`outlier-campaign-agent/0.2 (sentiment_miner; +https://outlier.ai)`) avoids Reddit 429 per Pitfall 3 in RESEARCH-V2.
- **LiteLLM call** — `OpenAI(base_url=config.LITELLM_BASE_URL, api_key=config.LITELLM_API_KEY)` exactly per `src/inmail_copy_writer.py:172-191`. Default model `anthropic/claude-haiku-4-5`; on a "model not found / invalid / unknown" exception the call retries once with `config.LITELLM_MODEL`.
- **System prompt** — embeds the CLAUDE.md vocabulary substitution table verbatim with explicit `NEVER "<banned>"` rules. The LLM is instructed to paraphrase theme labels into approved Outlier vocabulary while keeping `evidence_quotes` verbatim from the contributor input.
- **Vocabulary scrub** — defense-in-depth `_scrub_vocab()` regex pass runs over every theme label and every directive_for_brief before write. The unit test deliberately injects banned tokens via the LLM stub and asserts the scrub catches them.
- **Directive shaping** — sentiment + label keyword heuristic decides between `"Address: ..."`, `"Lean into: ..."`, and `"Avoid: ..."` directives. Themes whose labels mention `confus`, `unclear`, `broken`, `removed`, or `deprecated` get `"Avoid: ..."`; positive sentiment becomes `"Lean into: ..."`; default for negative/neutral is `"Address: ..."`.
- **PII safety** — Zendesk and Intercom fetchers never persist requester id, name, or email. Bodies are truncated to 800 chars in-memory and re-truncated to 400 chars when written to the raw dump. The on-disk artifacts contain only public review URLs, public Reddit/Discourse text, and theme summaries.
- **Failure isolation** — `run()` wraps each fetcher in try/except, records `sources_queried` vs `sources_skipped: [{source, reason}]`, and writes the output file even when only one source succeeded (or none — empty themes is a valid week).
- **Credential gating** — `fetch_zendesk()` and `fetch_intercom()` early-return `[]` and log a warning when their respective env vars are empty strings. Public sources still run.

## Deviations from Plan

None — plan executed exactly as written. The plan's task structure, file-modification list, vocabulary rules, PII rules, and acceptance criteria all matched the implemented behavior on the first pass.

A minor judgement call: the `_directive_for_theme` helper internally classifies `"Avoid"` themes via a small keyword tuple (`"confus"`, `"unclear"`, `"broken"`, `"removed"`, `"deprecated"`). The plan named the first three; `"removed"` and `"deprecated"` were added because they read as the same family of complaint and are mentioned in the brief-generator's section as features the agent should NOT frame ads around. This is documentation-consistent rather than a deviation.

## Authentication gates

None — the plan deliberately treats missing Zendesk + Intercom credentials as graceful skip-sources rather than auth gates. Live Zendesk and Intercom data will only flow once `ZENDESK_*` and `INTERCOM_ACCESS_TOKEN` env vars are set, but the weekly run still produces valid public-only output without them.

## Blockers

- `ZENDESK_SUBDOMAIN` / `ZENDESK_EMAIL` / `ZENDESK_API_TOKEN` env vars must be set for Zendesk source to activate. Without them, only public sources run (which is acceptable for the weekly cycle).
- `INTERCOM_ACCESS_TOKEN` env var must be set for Intercom source to activate. Same skip-with-warning behavior.
