# Phase 02.5 Plan 03: Experiment Scientist Agent & Test Directive System Summary

## One-Liner

Experiment backlog persistence + scientist agent: priority-ordered hypothesis queue with 20/80 test/baseline allocation feeding test directives to ad-creative-brief-generator.

## What Was Built

### Task 1: ExperimentBacklog persistence layer (src/memory.py)

`ExperimentBacklog` class managing hypothesis queue across process restarts:
- `load()` / `save()` — JSON persistence to `data/experiment_backlog.json`
- `add_hypothesis()` — deduplicates by `(cohort, angle, photo_subject)` key; auto-sorts by `priority_score` descending after each add
- `peek_next(n)` — returns top N pending experiments without modifying state
- `mark_running(key)` — transitions experiment to "running" when test starts
- `mark_completed(key, result)` — stores result dict for hypothesis validation
- `clear_pending()` — reset utility for testing/emergency
- Error handling: corrupted JSON falls back to empty backlog with `log.error()`
- `load_backlog(filepath)` convenience function for session startup

### Task 2: ExperimentScientistAgent (src/experiment_scientist_agent.py)

`ExperimentScientistAgent` class consolidating feedback into prioritized test queue:
- `ingest_feedback(hypotheses, competitor_intel)` — processes feedback_agent hypotheses + competitor intelligence
  - Priority score formula: `impact × confidence (0.9) × feasibility (0.9)`
  - Impact parsed from string like `"CTR+5%"` → `5.0`, defaults to `3.0` on parse failure
  - Competitor ideas matching existing cohort/angle get +0.5 priority boost
  - New competitor ideas added as fresh hypotheses with `priority_score=0.7`
- `generate_test_directive(cohort_name)` — returns test directive for creative brief generator
  - Returns `{test_allocation: 100}` (baseline) if no pending experiments for cohort
  - Returns `{test_allocation: 20}` (test variant) for top pending experiment; calls `mark_running()`
- `get_backlog_status()` — queue summary: total/pending/running/completed + next_to_test list

### Task 3: ad-creative-brief-generator spec updated (.claude/agents/ad-creative-brief-generator.md)

Added two sections:
- **Dependencies (Phase 2.5)** — documents experiment_scientist_agent and memory.py integration
- **Test Directive Integration (Phase 2.5 Feedback Loop)** — complete specification:
  - Baseline branch (80%): no test_directive or test_allocation=100 → standard generation, `experiment: false`
  - Test branch (20%): test_allocation=20 → use directive's angle + photo_subject, `experiment: true`
  - Angle-specific prompt modifications (A/B/C)
  - Output schema with new fields: `experiment`, `test_angle`, `test_photo`
  - Fallback behavior for invalid or missing test_directive
  - Example flow: test_directive input → brief output with experiment metadata
  - Tracking section: campaign metadata → feedback_agent measures → mark_completed()

## How to Use

```python
from src.memory import ExperimentBacklog
from src.experiment_scientist_agent import ExperimentScientistAgent

# 1. Load (or create) persistent backlog
backlog = ExperimentBacklog()  # loads from data/experiment_backlog.json

# 2. Ingest weekly feedback from feedback_agent + competitor_intel
agent = ExperimentScientistAgent(backlog)
agent.ingest_feedback(
    hypotheses=feedback_agent.analyze_creative_performance(),
    competitor_intel=json.load(open("data/competitor_intel/latest.json"))
)

# 3. Before each creative brief generation, get test directive for cohort
directive = agent.generate_test_directive("DATA_ANALYST")
# Returns: {cohort, angle, photo_subject, test_allocation (20 or 100), priority, expected_impact}

# 4. Pass directive to ad-creative-brief-generator
# If test_allocation=20: brief uses test angle + photo_subject, sets experiment=true
# If test_allocation=100: brief uses baseline generation, experiment=false

# 5. After campaign runs, close the loop
backlog.mark_completed(("DATA_ANALYST", "B", "spreadsheet"), result={"ctr": 0.08, "cpa": 12.5})
```

## Key Implementation Details

- **Priority scoring:** `priority_score = impact_float × 0.9 × 0.9`. For `"CTR+5%"` → score = 4.05. Capped at 10.0.
- **Deduplication:** Adding same `(cohort, angle, photo_subject)` triple twice is silently ignored (no-op)
- **Competitor boosting:** String ideas from `competitor_intel["experiment_ideas"]` try to match existing by `(cohort, angle)` and boost by +0.5 rather than creating duplicates
- **Backlog auto-sort:** Every `add_hypothesis()` call re-sorts the full list by `priority_score` descending
- **Test directive tracking:** `mark_running()` is called immediately when directive is issued to prevent double-scheduling the same experiment
- **Brief output fields:** `experiment` (bool), `test_angle` (str or null), `test_photo` (str or null) — enables feedback_agent to query campaigns WHERE experiment=true
