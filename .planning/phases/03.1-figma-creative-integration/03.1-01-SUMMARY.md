## Technical Notes

### photo_base64 Format
- **Input:** PNG file from Gemini (1-3 MB raw)
- **Processing:** `base64.b64encode(png_bytes).decode()`
- **Output:** `data:image/png;base64,{base64_string}`
- **Encoded size:** ~1.3× raw size (1.5 MB PNG → 1.9 MB encoded)
- **Claude context:** Acceptable for context window (~2 MB < 200K token budget)

### Agent Integration Path
```
dry_run.py (Stage 8)
  ├─ generate_midjourney_creative() → PNG file
  ├─ png_to_base64(png_path) → base64 string with data URI prefix
  ├─ Assemble agent_context with photo_base64
  └─ [Claude agent spawned]
      └─ outlier-creative-generator (Stage 8g)
          ├─ Receives photo_base64 in context
          ├─ Calls build_figma_layered_frame_js(photo_base64=...)
          └─ Creates 3 Figma frames with raster photo backgrounds
```

### Known Limitations
- Copy generation currently fails due to upstream LLM connectivity (not photo_base64 related)
- Stage C audienceCounts API requires LinkedIn MDP approval (gracefully bypasses to Stage B)
- Figma frame creation requires agent trigger (not automated in dry_run.py)
