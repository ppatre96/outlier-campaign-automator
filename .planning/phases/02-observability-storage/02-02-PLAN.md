# Drive Upload + Sheets Logging Fix

## Goal

Close the concrete code gap where `drive_url` is assigned in `main.py` (line 353) but silently dropped — it is never passed to `write_creative()` which has no parameter for it. After this plan the Drive URL will be persisted as a 5th column in the `Creatives` tab whenever `GDRIVE_ENABLED=true`.

## Background

Current call chain (main.py lines 348–367):
```python
drive_url = None
if config.GDRIVE_ENABLED:
    drive_url = upload_creative(png_path)   # returns webViewLink string

# ... later ...
sheets.write_creative(cohort._stg_id, cohort._stg_name, creative_urn)
# drive_url is NEVER passed — it is dropped here
```

Current `write_creative()` signature in `src/sheets.py` line 181:
```python
def write_creative(self, stg_id: str, creative_name: str, li_creative_id: str) -> None:
    ws = self._triggers.worksheet(config.CREATIVES_TAB)
    ws.append_row([
        stg_id,
        creative_name,
        li_creative_id,
        datetime.utcnow().isoformat(timespec="seconds") + "Z",
    ], value_input_option="RAW")
```

`src/gdrive.py` is fully implemented and correct — no changes needed there.
`GDRIVE_FOLDER_ID` is currently a personal Drive folder ID — must be replaced with a Shared Drive folder ID (external admin step).

## Tasks

### Task 1 — Add `drive_url` parameter to `write_creative()` in `src/sheets.py`

- **File:** `src/sheets.py`
- **Change:** Update the `write_creative` method signature at line 181 and its `append_row` call:

  **Before:**
  ```python
  def write_creative(self, stg_id: str, creative_name: str, li_creative_id: str) -> None:
      ws = self._triggers.worksheet(config.CREATIVES_TAB)
      ws.append_row([
          stg_id,
          creative_name,
          li_creative_id,
          datetime.utcnow().isoformat(timespec="seconds") + "Z",
      ], value_input_option="RAW")
      log.info("Wrote creative %s → %s", stg_id, creative_name)
  ```

  **After:**
  ```python
  def write_creative(
      self,
      stg_id: str,
      creative_name: str,
      li_creative_id: str,
      drive_url: str = "",
  ) -> None:
      ws = self._triggers.worksheet(config.CREATIVES_TAB)
      ws.append_row([
          stg_id,
          creative_name,
          li_creative_id,
          datetime.utcnow().isoformat(timespec="seconds") + "Z",
          drive_url,
      ], value_input_option="RAW")
      log.info("Wrote creative %s → %s (drive_url=%s)", stg_id, creative_name, drive_url or "none")
  ```

  The `drive_url` parameter defaults to `""` so all existing call sites that do not pass it remain valid without modification.

### Task 3 — Document Shared Drive setup in README.md

- **File:** `README.md`

```markdown
## Google Drive Setup (DATA-01)

Generated PNG creatives are uploaded to Google Drive when `GDRIVE_ENABLED=true`.
The service account cannot upload to a personal Drive folder (quota restriction).
A Google Workspace **Shared Drive** is required.

### Admin steps (one-time, requires Google Workspace admin access)

1. In Google Drive, click **New → Shared drive** → name it `Outlier Campaign Creatives`.
2. Open the new Shared Drive → click the settings gear → **Manage members**.
3. Add `outlier-sheets-agent@outlier-campaign-agent.iam.gserviceaccount.com` as **Content Manager**.
4. Copy the Shared Drive folder ID from its URL:
   `https://drive.google.com/drive/folders/<FOLDER_ID>`
5. Set in `.env`:
   ```
   GDRIVE_ENABLED=true
   GDRIVE_FOLDER_ID=<FOLDER_ID>
