# Lifecycle Monitor Slack Notification + Cron Verification

## Goal

The campaign lifecycle monitor writes verdicts to the `Monitor` tab in Google Sheets but never posts them to Slack. The Monday weekly report is the single delivery touchpoint. This plan adds a `read_monitor_summary()` function that reads the most recent monitor rows from the `Monitor` tab and returns a formatted text block, then wires that block into `post_weekly_reports.py` so verdicts appear in the Monday DM alongside the InMail and static reports.

## Background

`run_monitor()` in `main.py` lines 573–655 does everything correctly except Slack delivery.
`write_monitor_results()` in `src/campaign_monitor.py` lines 230–257 appends rows to the `Monitor` tab with columns: `Date, STG ID, Campaign ID, Flow ID, Pass Rate %, Cohort Avg %, Verdict`.
`scripts/post_weekly_reports.py` currently calls only `inmail_report()` and `static_report()` — the `Monitor` tab is not read.
`_post_to_slack()` in `post_weekly_reports.py` will be updated by plan 02-01 to use the Bot Token path. This plan depends on that change being in place (wave: 2, depends_on: [02-01-PLAN]).

### Task 1 — Add `read_monitor_summary()` to `src/campaign_monitor.py`

```python
def read_monitor_summary(sheets, max_rows: int = 20) -> str:
    """
    Read the most recent rows from the 'Monitor' tab and return a
    Slack-ready summary string.

    Returns an empty string if the Monitor tab does not exist or has no data
    rows (i.e. monitor has never been run).

    Args:
        sheets:   SheetsClient instance (already authenticated).
        max_rows: Maximum number of recent rows to include in the summary.
                  Defaults to 20 (covers up to 20 campaigns per monitor run).

    Returns:
        Formatted text block, e.g.:
            Campaign Monitor (2026-04-21):
              KEEP  : skills__diagnosis__healthcare (pass 34.2%, avg 28.1%)
              PAUSE : skills__engineering__electrical (pass 18.5%, avg 28.1%)
            1 campaign flagged for review.
        Or empty string if no data is available.
    """
    try:
        ws = sheets._triggers.worksheet("Monitor")
    except Exception:
        log.info("Monitor tab not found — skipping monitor summary")
        return ""

    rows = ws.get_all_values()
    if len(rows) < 2:
        log.info("Monitor tab has no data rows")
        return ""

    # Header: Date, STG ID, Campaign ID, Flow ID, Pass Rate %, Cohort Avg %, Verdict
    data_rows = rows[1:]  # skip header
    # Take the most recent max_rows
    recent = data_rows[-max_rows:]

    # Group by date — use the date of the last row as the report date
    report_date = recent[-1][0] if recent else "unknown"

    lines = [f"*Campaign Monitor ({report_date}):*"]
    pause_count = 0
    for row in recent:
        if len(row) < 7:
            continue
        _date, stg_id, campaign_id, flow_id, pass_rate, cohort_avg, verdict = row[:7]
        label = stg_id or campaign_id or flow_id or "unknown"
        lines.append(
            f"  {verdict:<10}: {label} "
            f"(progress {pass_rate}%, avg {cohort_avg}%)"
        )
        if verdict.upper() == "PAUSE":
            pause_count += 1

    if pause_count:
        lines.append(f"{pause_count} campaign(s) flagged for review — check Monitor tab.")
    else:
        lines.append("All monitored campaigns within progress thresholds.")

    return "\n".join(lines)
```

  Note: The word "pass rate" in the existing column header and existing code uses the term "pass_rate" — this is an internal analytics term (screening pass rate), not user-facing copy. It is acceptable in the Sheets tab. The Slack-facing label uses "progress" per CLAUDE.md vocabulary rules (do not say "performance").

## Notes

- The Slack monitor message uses "progress" (not "performance" or "pass rate") per CLAUDE.md vocabulary rules when referring to campaign metrics visible to users. Internal analytics fields (column headers in the Monitor tab, Python variable names) are not user-facing and may retain the analytics terminology.
- `read_monitor_summary()` takes a `sheets` argument rather than constructing its own `SheetsClient` to avoid a second authentication round-trip when `post_weekly_reports.py` already creates a client.
- The `Monitor` tab header row is: `Date, STG ID, Campaign ID, Flow ID, Pass Rate %, Cohort Avg %, Verdict` — row index 0 in `ws.get_all_values()`. The function skips this row with `rows[1:]`.
- This plan depends on 02-01-PLAN being complete because `_post_to_slack()` must be the Bot Token version. If the Bot Token is not yet configured, the monitor summary will be generated but not delivered — which is the correct graceful degradation (error logged, script exits normally).
- Do NOT post monitor results from within `run_monitor()` in `main.py`. The Monday Slack report (`post_weekly_reports.py`) is the single delivery channel, keeping the DM timeline clean and timing predictable.
