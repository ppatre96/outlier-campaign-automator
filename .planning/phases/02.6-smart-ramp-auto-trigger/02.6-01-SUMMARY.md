# Phase 02.6 Plan 01: Smart Ramp Poller + State File + Edit Detection Summary

**One-liner:** Built the launchd-driven Smart Ramp poller scaffolding — content-hash signature, atomic state IO, filelock concurrency guard, word-boundary test-ramp filter, and 5-failure escalation gate — with the pipeline call left as a STUB for Plan 02.

## What Was Built

`scripts/smart_ramp_poller.py` (~330 LOC) wires every primitive Phase 2.5 V2's `weekly_feedback_loop.py` already shipped, adapted for ramp polling:

- **load_dotenv-first** import ordering so `config.py`'s env-time reads see `.env` values before any Smart Ramp constant is consumed (Pitfall 3 from Phase 2.5 V2 RESEARCH-V2 §Pitfall 6).
- **Atomic state write** via `tempfile.mkstemp` + `os.fsync` + `os.replace`. A simulated SIGKILL between write and rename leaves `data/processed_ramps.json` either at pre-write content or post-write content — never partial JSON, no `.tmp` leftovers.
- **Content-hash signature** (`compute_signature`) — sha256 over `json.dumps([asdict(c) for c in sorted(cohorts, key=id)], sort_keys=True) + "\n" + summary + "\n" + updated_at`. Deterministic across re-fetches AND cohort-list permutations. Matches the locked algorithm in `02.6-CONTEXT.md` exactly so the 8 pre-seeded ramps in `data/processed_ramps.json` (committed `fc3ad60` on 2026-04-27) classify as `noop` on the first live poll instead of re-firing all 8.
- **Edit detection** (`_classify_action` / `_mark_superseded`) — `prior == None → "new"`, signature match → `"noop"`, signature differ → `"edit"`. On `"edit"`, the live entry is archived to `state["ramp_versions"][f"{id}_v{prior}"]` with `superseded=True`, version is bumped, and the live entry is overwritten cleanly with the new content.
- **Test-ramp filter** (`_should_skip_test_ramp`) — `re.compile(config.SMART_RAMP_TEST_REQUESTER_PATTERN, re.IGNORECASE).search(name)`. Default pattern is `r"\btest\b"` (word-boundary). "Quintin Au Test" → filtered. "Christopher Testov" → NOT filtered (substring match would have wrongly skipped it).
- **filelock concurrency guard** — `FileLock(str(LOCK_PATH), timeout=5)` wraps `run_once`. Contention raises `Timeout`, which `main` catches, logs `"previous poll still running — exiting cleanly"`, and returns 0 (NOT raise; NOT exit non-zero).
- **5-failure escalation gate** — `process_ramp` increments `consecutive_failures` on each failure. When the count hits `config.SMART_RAMP_FAILURE_THRESHOLD` (default 5), `escalation_dm_sent` flips to `True` and `_should_block_for_escalation` short-circuits subsequent polls until the counter resets to 0 (manual file edit per CONTEXT.md). The actual escalation DM lives in Plan 03 — this plan only flips the flag.
- **`run_ramp_pipeline` STUB** — returns a deterministic mock-success dict with `ok=True` and empty campaign/creative lists. Plan 02 will replace the body with `from main import run_launch_for_ramp`. Stub is explicitly commented `# STUB: replaced by Plan 02 ...` so the verifier flags it.
- **CLI flags** — `--once`, `--ramp-id <id>`, `--dry-run`. `--ramp-id` bypasses the signature noop and force-processes a single ramp; `--dry-run` runs everything except state write.

# Dry run (no state write, no Slack, but everything else fires for inspection)
