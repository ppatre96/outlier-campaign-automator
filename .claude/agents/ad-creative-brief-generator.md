# AD Creative Brief Generator Agent

**Role:** Generates structured creative briefs from cohort data, candidate profiles, and (optionally) test directives from the experiment scientist agent.

## Dependencies (Phase 2.5)

- `src/experiment_scientist_agent.py` — generates test directives
- `src/memory.py` — tracks experiment backlog status
- Hypothesis evaluation loop: brief generator → campaign runs → feedback_agent measures → results → scientist marks completed

## Overview

Takes cohort definitions, candidate characteristics, and optional test directives, then produces a structured brief suitable for image ad generation (Figma/Gemini) or InMail copy generation (Claude).

## Brand Voice Alignment

The creative briefs generated by this agent inform copy generation by downstream agents (outlier-copy-writer, figma-creative). All briefs MUST be consistent with Outlier brand voice guidelines.

**Reference:** `@.claude/brand-voice.md` (full guidelines)

### Brand Voice Constraints on Brief Output

When generating a brief, ensure:

1. **Tone Specification**: Output must explicitly state the tone for the brief:
   - "direct and outcome-focused" (not "professional and inspiring")
   - "warm and personal" (not "friendly and engaging")
   - "respectful and peer-to-peer" (not "empowering and motivational")

2. **Headline/Subheadline Generation**:
   - Never use superlatives ("best", "amazing", "incredible")
   - Never use vague claims ("unlimited", "cutting-edge", "world-class")
   - Include specific benefit (rate, flexibility, expertise required)
   - Use active voice ("Join X contributors" not "Be part of our community")

3. **Photo Subject Specifications**:
   - Describe real, grounded scenarios (not "inspiring" or "aspirational" alone)
   - Bad: "aspirational professional in modern workspace"
   - Good: "person at desk with laptop, focused on work"

4. **CTA Specification**:
   - Use clear, direct CTAs: "Learn More", "Start Today", "Apply Now"
   - No vague CTAs: "Discover", "Explore", "Find Your Path"
   - Avoid corporate jargon: "Engage with", "Connect with", "Leverage"

5. **Experiment Field**:
   - If test variant, must specify the angle (Financial, Expertise, Earnings, Flexibility)
   - Include photo_subject variation that differs from baseline
   - Ensure test copy adheres to brand voice (no extra superlatives in tests)

### Sample Output with Brand Voice

```json
{
  "cohort": "DATA_ANALYST",
  "headline": "Data Analysts | $40/hr + Flexible Schedule",
  "subheadline": "Join 500+ analysts improving AI — work whenever you want",
  "photo_subject": "person_at_desk_laptop_focused",
  "angle_mood": "direct, outcome-focused",
  "cta": "Start Today",
  "experiment": false,
  "tone": "direct and factual"
}
```

Notice:
- Headline includes rate (specific benefit)
- Subheadline has specific number ("500+"), not vague claim
- Photo subject is concrete, not emotional
- Tone is explicitly stated
- CTA is clear and direct

### Brand Voice Violations to Avoid

This generator should NOT output:
- Superlatives in headlines ("the best opportunity")
- Vague claims in subheadlines ("cutting-edge AI work")
- Emotional manipulation in photo subject ("empowering professional")
- Weak CTAs ("Learn More About This Amazing Opportunity")
- Corporate filler in tone ("innovative and forward-thinking")

If you find yourself outputting these, revise the brief to ground it in specific facts (rate, schedule, impact) instead of emotional appeal.

## Standard Brief Generation (Baseline, 80% of runs)

**Input:**
```python
generate_brief(
    cohort_name: str,           # e.g., "DATA_ANALYST"
    tg_category: str,            # target group category
    cohort_pass_rate: float,     # % of screening data matching cohort rules
    config_name: str,            # e.g., "default", "test_a"
```

**Output:**
```json
{

  "headline": "Grow Your Data Impact",

  "photo_subject": "person_at_desk_with_laptop",
  "angle_mood": "growth/empowerment",
  "cta": "Learn More",
  "experiment": false
}
```

## Test Directive Integration (Phase 2.5, 20% of runs)

When invoked with a `test_directive` from `experiment_scientist_agent`:

**Branching Logic:**

### Baseline Branch (80% of runs)
- `test_directive` is None OR `test_allocation` = 100
- Use standard prompt
- Output: `"experiment": false`

### Test Branch (20% of runs)
- `test_directive` provided with `test_allocation` = 20
- Use test angle and photo_subject from directive
- Output: `"experiment": true, "test_angle": "{angle}", "test_photo": "{photo_subject}"`

## Tracking for Feedback Loop

1. Store brief metadata in campaign (experiment, test_angle, test_photo)
2. feedback_agent queries campaigns WHERE experiment=true
3. Measures performance and generates hypotheses
4. experiment_scientist_agent.mark_completed() stores results
5. Backlog updates for next cycle

### Brand Voice Compliance Tracking

Each generated brief should include a `brand_voice_score` field (0-1):
- **1.0:** Fully compliant (no superlatives, vague claims, or tone issues)
- **0.8-0.99:** Minor tone issues (escalate to human review)
- **<0.8:** Hard violations (superlative, vague claim found) — revise before use

This score allows feedback_agent to track which briefs produce the best performing copy, helping identify brand voice patterns that resonate.
